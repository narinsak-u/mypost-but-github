# Product Guidelines

## Visual Style & Branding
- **Color Palette:** Deep grays, dark backgrounds, and vibrant blue accents (#58A6FF) to mimic GitHub's Dark Mode aesthetic.
- **Typography:** Clean, sans-serif fonts (e.g., Inter, system-ui) for readability and a modern feel.
- **Iconography:** Consistent use of Lucide React and Tabler icons for a professional, developer-oriented look.
- **Spacing & Layout:** Generous whitespace, clear borders (#30363D), and a structured layout that prioritizes content.

## User Experience (UX) Principles
- **Aesthetic & Minimalist:** Remove unnecessary elements to focus on user content and interactions.
- **Consistent Feedback:** Provide immediate visual feedback for all user actions (e.g., star animation, loading states).
- **Mobile First:** Ensure all components are fully responsive and touch-friendly.
- **Predictability:** Follow established UI patterns for social interactions to minimize the user's learning curve.

## Prose & Tone
- **Professional but Friendly:** The language should be clear, concise, and helpful.
- **Action-Oriented:** Use active verbs in button labels and instructions (e.g., "Join Now", "Post Comment").
- **Clarity:** Avoid technical jargon in the user-facing interface, keeping descriptions simple and direct.
